# inbox

Разобрать инбокс: превратить сырые записи из `_inbox/` в задачи и очистить инбокс.

## Vault location

Корень текущего репозитория. Инбокс: `_inbox/*.md`, каждый файл — одна запись:

```
---
created: <ISO-дата>
project: <slug или отсутствует>
tags: [inbox]
---

<текст мысли>
```

## Steps

1. **Подтянуть свежее**: `git pull --rebase --autostash` (записи могут приходить с других устройств).
2. **Собрать записи**: все `_inbox/*.md` (кроме `.gitkeep`). Пусто → сказать «инбокс пуст» и закончить.
3. **Каждую запись → задача** (по логике навыка `backlog`):
   - Проект из frontmatter `project`; нет — определить из текста; сомнение → в «Вопросы» задачи, не в чат
   - Сквозной `id` (max + 1 на каждую новую)
   - RICE + est_days, все разделы по шаблону
   - В «Заметки»: `Из инбокса <created>: "<оригинальный текст>"`
   - Запись-не-задача (заметка, ссылка, идея «на подумать») → вынести в карточку проекта или роадмап, тоже удалить из инбокса
   - Запись-поручение («проверь X») → по возможности выполнить сразу и зафиксировать результат
4. **Удалить разобранный файл** из `_inbox/`.
5. **Commit + push** (если vault — git-репо).
6. **Отчитаться** списком: запись → созданная задача (проект, файл, RICE) или что сделано; где были сомнения.

## Important rules

- Не оставлять инбокс наполовину разобранным: каждая запись либо стала задачей/идеей и удалена, либо явно пропущена с объяснением
- Не редактировать `_inbox/` в обход цикла (write → delete → commit → push)
- Содержимое задач — на русском
