# taskvault — правила для агента

Это vault — персистентная память о проектах и задачах. Пользователь не редактирует файлы вручную: **все изменения вносишь ты, сразу по ходу разговора**. Полная методология — в `_rules.md` (прочитай при первой работе с vault).

Корень vault = корень этого репозитория.

## Главное

- Начало разговора о проекте → прочитай `projects/<проект>/<проект>.md`; тема неясна → `_dashboard.md`.
- Появился новый факт, решение, задача или статус → обнови соответствующий файл **немедленно**, не дожидаясь конца разговора.
- Каждое значимое действие отражается в задаче: пишешь код → задача в `doing`/`done`; обсуждается будущая фича → задача `todo`. Нельзя завершить работу, не обновив статус.
- Задачи создавай из `_templates/task.md` со сквозным `id` (max по `projects/*/tasks/` + 1) и RICE-оценкой — детали в навыке `backlog`.
- Если vault — git-репозиторий: после содержательных правок `git add -A && git commit && git push`.

## Навыки

- `sync` — загрузить контекст всех проектов в сессию
- `backlog` — завести задачу (RICE, номер тикета)
- `inbox` — разобрать `_inbox/` в задачи

## Чего не делать

- Не удалять информацию без явной просьбы пользователя
- Не менять структуру папок и заголовков
- Не задавать в чате вопросы по задаче — записывай их в раздел «Вопросы» файла задачи
