# recap — Дайджест

Собрать **Дайджест** за период: не дамп задач, а умный нарратив — что сделано и какой с этого эффект, что свернули и почему, какие темы и куда движется работа. Для саморефлексии, 360-ревью и отчётов. По умолчанию дайджест **сохраняется файлом** в vault (это память о прогрессе, она должна пережить сессию). Самодостаточно: можно дать любому агенту.

## Когда генерировать

- **Авто в начале сессии.** Навык `sync` проверяет, не закрылся ли период без дайджеста (см. ниже «Каденция»). Если закрылся — **сгенерируй и закоммить дайджест сам**, в начале сессии, без отдельной просьбы.
- **По команде** — «дайджест за месяц», «итоги недели», «что я сделал за…».
- **На прощание** — пользователь уходит («на сегодня всё», «до завтра») → дай дайджест дня (день можно не сохранять, если периодического смысла нет; недельный+ — сохраняй).

## Каденция и периоды

Дайджесты — по грядке. Периоды и их идентификаторы файлов:

- **Неделя** — ISO-неделя, `<bed>/digests/2026-W26.md`
- **Месяц** — `<bed>/digests/2026-06.md`
- **Полгода** — H1 (янв–июн) / H2 (июл–дек), `<bed>/digests/2026-H1.md`

Путь: корневая грядка — `digests/<период>.md` в корне vault; прочие — `beds/<slug>/digests/<период>.md`.

**Детекция закрытия периода (для sync):** период считается закрытым, когда календарно завершился (прошёл) и в `<bed>/digests/` нет файла с его идентификатором. Сгенерируй недостающие за уже завершившиеся периоды (не за текущий незавершённый). Не плоди дубли — есть файл, не перезаписывай без явной просьбы.

## Что собрать (источники)

1. **Закрытые задачи** (`status: done`) с датой закрытия в периоде (`closed_at`, иначе git-дата коммита, выставившего `done`). Группировать по проектам.
2. **Свёрнутое** (`status: cancelled`) в периоде — с причиной (тело задачи / `close_note`): «что передумали делать» ценно для ретроспективы.
3. **Логи сессий** («## Что сделано в сессии <дата>» в карточках проектов) за период — детали, контекст, формулировки эффекта.
4. (опц.) Что в работе (`doing`) и крупное запланированное — куда движемся.

## Как писать дайджест (это и есть «по-умному»)

Не перечисление тикетов, а **синтез**. Структура файла:

```markdown
---
period: 2026-06
type: month            # week | month | half
range: 2026-06-01..2026-06-30
bed: daily
generated: 2026-07-01
---

# Дайджест — июнь 2026

<Абзац-итог: главное за период одним связным куском — что изменилось, куда продвинулись.>

## Хайлайты
- <2–4 самых значимых результата — с эффектом, а не «закоммитил X». Что стало возможно/лучше.>

## По темам
- **<тема/проект>** — что сделано и зачем, человеческим языком (outcome). 2–5 пунктов.

## Свернули
- <что отменили и почему — осознанные «нет» тоже результат.>

## Куда дальше
- <1–3 пункта: что в работе или напрашивается следующим.>
```

Принципы текста:
- **Эффект, не активность.** «Развернул ручной трекер — личные задачи теперь живут отдельно от рабочих» вместо «закрыл 12 задач».
- **Группируй по смыслу**, не по дате закрытия. Близкие задачи сворачивай в один тезис.
- **Честно** — только реально закрытое/свёрнутое; не приукрашивать, не выдумывать эффект.
- **Стиль** — styleguide Огорода: лаконично, без воды и нейро-слопа, активный залог, финальный проход-вычёркивание.

## 360-ревью (по запросу)

Если просят «для ревью»/«по компетенциям» — добавь раздел, где результаты сгруппированы по компетенциям: delivery (довёл до прода), ownership (закрыл без напоминаний), качество (верификация/фиксы), влияние на продукт/метрику, инициатива. Каждую — 1–2 конкретными результатами периода.

## Сохранение и коммит

- Запиши файл в `<bed>/digests/<период>.md` (создай папку при необходимости).
- Если vault — git-репо: закоммить и запушь (дайджест — артефакт, дашборд его покажет).
- Существующий дайджест периода **не перезаписывай** без явной просьбы (можно дополнить, отметив правку).

## Правила

- Сохранение — поведение **по умолчанию** для недельных и крупнее (день — по ситуации).
- Чтение источников не меняет задачи; единственная запись — сам файл дайджеста.
- Язык — русский.

## Самопроверка

- [ ] Собраны done + cancelled за период (по дате закрытия) и логи сессий
- [ ] Дайджест — нарратив с эффектом, а не список задач; есть «свернули» и «куда дальше»
- [ ] Файл сохранён в `<bed>/digests/<период>.md` с frontmatter, закоммичен
- [ ] Не перезаписан существующий дайджест без просьбы; без приукрашивания
